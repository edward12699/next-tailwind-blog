MDX 是一种文件格式，它允许你在 Markdown 文档中编写 JSX，并且可以导入 React 组件。这意味着你可以在你的文档中使用 React 组件，这在写文档或者创建复杂的内容时非常有用。

MDX 的语法非常接近于标准的 Markdown 语法，但是它增加了一些额外的功能。例如，你可以像在 JSX 中那样使用大括号 `{}` 来插入 JavaScript 表达式。你也可以像在 JSX 中那样使用标签来包含和渲染 React 组件。

这是一个简单的 MDX 文件的例子：

```js
# Hello, MDX

You can **render** Markdown and JSX in the same file!

<YourReactComponent />

Here's a JavaScript expression: {1 + 1}
```

在这个例子中，`<YourReactComponent />` 就是一个 React 组件，它会像在任何其他的 React 应用中一样被渲染。`{1 + 1}` 是一个 JavaScript 表达式，它会被求值并插入到文档中。

MDX 可以在许多不同的环境中使用，包括但不限于 Gatsby, Next.js, Create React App, VuePress, Docz 等等。它是一种强大的工具，让你可以将编程和写作结合在一起，创建更加动态和互动的内容。

2. `<img src="url-to-image" alt="alt text" />`


## 关于强化样式

由于 MDX 支持 JSX，你可以直接在 MDX 中使用内联样式或者引入 CSS 样式表来控制颜色和其他样式属性

1. 直接内联 （不推荐，如果用的是react框架或者其他，有可能会导致渲染报错，且无法复用）

*`<p style={{ color: 'red' }}>This is a paragraph with red color.</p>`*


2. 使用components(推荐)

```js
//ColoredStrong.JSX

import clsx from 'clsx'
const variantStyles = {
  primary:
    'font-semibold text-emerald-500',
  secondary:
    'bg-zinc-50 font-medium text-zinc-900   ',
}

// colored strong
export function CS({ variant = 'primary', className, href, ...props }) {
  className = clsx(
    'inline-flex items-center gap-2 justify-center rounded-md  text-sm outline-offset-2 transition ',
    variantStyles[variant],
    className
  )

  return <span className={className} {...props} ></span>
}
```

```js
// mdx
import {CS} from '@/components/mdx-style/ColoredStrong'
<CS>xxx</CS>
```
