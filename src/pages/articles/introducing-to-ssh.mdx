SSH 是 "Secure Shell" 的缩写，它是一种网络协议，用于通过不安全的网络进行安全远程登录和其他安全网络服务。SSH 通过在两台计算机之间建立一个安全的隧道来工作，所有通过这个隧道的数据都会被加密，以防止在传输过程中被窃取或篡改

这里用几个常见的例子来说明ssh的过程

##  git 使用ssh认证的数据传输过程

1. **生成 SSH 密钥对**：用户在自己的机器上生成一个 SSH 密钥对，包括一个公钥和一个私钥。私钥被安全地存储在用户的机器上，不应该被分享或公开。公钥可以被公开分享，它将被上传到 Git 服务器。

2. **上传公钥到 Git 服务器**：用户将生成的公钥上传到 Git 服务器（例如 GitHub、GitLab 或 Bitbucket）。

3. **进行 SSH 连接**：当用户尝试通过 SSH 连接到 Git 服务器时，服务器会发送一个随机的挑战消息。然后，用户的 SSH 客户端会使用用户的私钥对这个消息进行加密，并将加密后的消息发送回服务器。

4. **服务器验证**：Git 服务器使用存储在服务器上的用户公钥来解密接收到的消息。如果解密的消息与最初的挑战消息匹配，服务器就知道用户拥有与公钥配对的私钥，因此认为用户是合法的。

5. **连接建立**：一旦服务器验证了用户的身份，就会建立 SSH 连接，用户可以开始执行 Git 操作（如拉取、推送等）。

## ssh  登陆服务器


1. **生成 SSH 密钥对**：用户在自己的机器上生成一个 SSH 密钥对，包括一个公钥和一个私钥。私钥被安全地存储在用户的机器上，不应该被分享或公开。公钥可以被公开分享。

2. **上传公钥到远程服务器**：用户将生成的公钥上传到远程服务器。这通常通过 scp 或者 ssh-copy-id 命令完成，或者直接将公钥内容复制并粘贴到远程服务器的 `~/.ssh/authorized_keys` 文件中。

3. **发起 SSH 连接请求**：用户在本地机器上运行 SSH 命令，并指定要连接的远程服务器和用户名。例如，`ssh username@remote_server`。

4. **服务器确认公钥**：远程服务器会检查接收到的 SSH 请求是否来自已知的公钥。它通过查找 `~/.ssh/authorized_keys` 文件来完成这个任务。如果文件中有一个与用户公钥匹配的条目，服务器就知道这是一个合法的请求。

5. **进行密钥挑战**：然后，服务器将生成一个随机数，并使用用户的公钥进行加密，然后将加密后的随机数发送回用户的 SSH 客户端。

6. **客户端解密挑战**：用户的 SSH 客户端使用私钥解密接收到的加密随机数，然后将解密后的随机数发送回服务器。

7. **服务器验证**：服务器对比客户端发回的随机数与最初生成的随机数是否一致。如果一致，服务器就认为客户端是合法的。

8. **建立连接**：一旦服务器验证了用户的身份，就会建立 SSH 连接，用户可以开始在远程服务器上执行命令。

这个过程确保了即使有人能嗅探到网络流量，他们也不能获得足够的信息来伪装为合法用户，因为他们没有用户的私钥来解密服务器的随机数挑战。

另外，使用pem 登陆服务器的原理也是ssh
`ssh -i /path/to/your/key.pem username@remote_server`
当你使用PEM文件进行SSH登录时，服务器将使用与PEM文件中的私钥匹配的公钥进行身份验证。
